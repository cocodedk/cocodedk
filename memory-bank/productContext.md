# Product Context

## Purpose

Cocode.dk serves as a digital business card and service showcase for a cybersecurity and software development consultancy. The purpose of this website is to:

1. Present the services offered in an engaging and interactive way
2. Demonstrate technical competence through the implementation itself
3. Provide visitors with essential information about the business
4. Protect contact information from automated scraping
5. Offer an accessible experience across different languages and devices

## Target Audience

- **Primary**: Potential clients seeking cybersecurity and software development services
- **Secondary**: Technical professionals evaluating the company's technical capabilities
- **Tertiary**: Business partners looking for collaboration opportunities
- **Edge case**: Talent scouts and potential employees exploring the company

The audience is expected to be technically savvy, with most visitors having experience in the IT industry. However, the site should be approachable for business stakeholders without strong technical backgrounds.

## User Experience Goals

### Engagement
- Create an immediate "wow" effect with the interactive visualization
- Use animations and visual feedback to maintain interest
- Keep loading time minimal to prevent bounce-rate
- Preserve clean, distraction-free information delivery

### Information Architecture
- Present information in an intuitive, discoverable way
- Use the visualization as a navigation metaphor
- Progressively disclose information based on user interest
- Protect sensitive information while keeping it accessible to legitimate users

### Accessibility
- Support multiple languages (currently 11)
- Ensure keyboard navigation for all features
- Follow ARIA best practices
- Maintain appropriate contrast for readability
- Provide alternative access to information when needed

### Visual Design Direction
- Modern, tech-focused aesthetic
- Matrix-inspired animations for the title
- Purple gradient background for depth and visual interest
- Node category colors that stand out while maintaining harmony
- Glass-morphism effects for UI containers
- Glowing accents for selected or hovered elements
- Flag icons for intuitive language selection
- Responsive design adapting to different device sizes

### Security and Privacy
- Obfuscate contact details from automated scraping
- Implement human verification for sensitive information access
- No tracking or analytics scripts
- No cookies or local storage (except language preference)
- No third-party scripts or resources

## Success Metrics

The product will be considered successful if:

1. Conversion rate from site visitor to inquiry increases
2. Users explore multiple nodes in the visualization
3. Users can find contact information when needed
4. The site functions properly across devices
5. Load time remains under 2 seconds
6. Automated scraping attempts are unsuccessful

## Constraints

- Single-page architecture
- No backend services
- No third-party libraries or frameworks
- Emphasis on clean code and maintainability
- Support for modern browsers only
